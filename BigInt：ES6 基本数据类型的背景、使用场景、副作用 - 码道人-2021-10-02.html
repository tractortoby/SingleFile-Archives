BigInt：ES6 基本数据类型的背景、使用场景、副作用

nisuoer·10 天前
举报

背景

早在 2019-9-30 BigInt 草案就有了，BigInt 将成为 Javascript 语言内置特性。为了解决 Javascript 能表达的最大整数只有 2^53 - 1 的问题（仅支持 Number 一种数字类型，标准为 IEEE754）。

写过 REST 接口的同学都知道，Java 等语言的长整类型（long）可以表达 2^63 - 1 的数值，所以在序列化时一些 ID 值必须转换为字符串再传递，不能是纯数值，否则有数值截断的问题，Java 常见的写法：

import com.fasterxml.jackson.databind.annotation.JsonDeserialize;
import com.fasterxml.jackson.databind.annotation.JsonSerialize;
import com.fasterxml.jackson.databind.ser.std.ToStringSerializer;

public class TestEntity {

    @JsonSerialize(using = ToStringSerializer.class)
    @JsonDeserialize(as = Long.class)
    private Long id;
    // other fields......
}

配合 Jackson 序列化库实现 ID 字符串转换。ID 并不需要进行计算，但如果 Javascript 运行在服务端，IEEE754 显然就不能满足一些计算需求了（比如科学计算场景），BigInt 的出现就是必须的了。

兼容性

目前浏览器支持率 91.23%，一些国产浏览器不支持 BigInt，兼容性较差，需要使用 polyfill 支持，比如这个：big-integer。

安装：

npm install big-integer

polyfill 使用：

var bigInt = require("big-integer");
var zero = bigInt();
var ninetyThree = bigInt(93);
var largeNumber = bigInt("756435643634734534563423785906736546456235345");
var googol = bigInt("1e100");
var bigNumber = bigInt(largeNumber);

var maximumByte = bigInt("FF", 16);
var fiftyFiveGoogol = bigInt("<55>0", googol);

使用

BigInt 对象自带两个静态方法：

  • BigInt.asIntN：转换为有符号整数
  • BigInt.asUintN：转换为无符号整数

实例方法：

  • toLocaleString：转换为特定地区语言的字符串
  • toString：转换为普通字符串
  • valueOf：获取 BigInt 对象的基元值

可以直接在常规 Number 后面增加一个字符 n 表示一个 BigInt，比如：

const testBigInt = 10086n;

类型信息：

typeof 10086n === 'bigint'; // true
typeof BigInt('10086') === 'bigint'; // true

其他的数值操作和 Number 类似，可以参考 MDN 文档，这里不再赘述，相关副作用可以参考后面的表格，这里讲讲 BigInt 的几个使用场景（主要是服务端场景）。

Long 类型分布式 ID

分布式 ID 是一个很常见的需求，需要保持数据库主键唯一性的同时带有自增的属性，同时还要支持并发生成，常见的实现方案有 twitter snowflake 方案（63bit）：

A distributed unique ID inspired by Twitter's Snowflake.
ID data bits:
+----------------------+----------------+-----------+
|  delta millisecond   | worker node id | sequence  |
+----------------------+----------------+-----------+
|        41bits        |     10bits     |   12bits  |
+----------------------+----------------+-----------+

可以看到 Javascript 的 Number 类型不能满足需求，BigInt 可以用来表示一个大整数分布式 ID。

高精度时间戳

精确到纳秒级别的时间戳很常见，经常用来记录特定程序的执行事件，以便进行性能分析，BigInt 可以表示高精度时间戳，在后端性能分析、性能调优中发挥作用。

科学计算、金融分析领域

大数值在科学计算或金融领域非常常见，甚至可以用来作为 BigDecimal 的一种实现（Javascript 没有 BigDecimal 类型），解决著名的 0.10 + 0.20 ！== 0.30 的问题，毕竟算钱算错就出事了。

副作用

BigInt 的使用有很多注意点，每一步数值操作都要检查下是否会有副作用的产生，是否在控制范围内。相关数值操作支持程度可以对照下表：

操作 是否支持
单目 (+) 运算符 N
+、*、-、**、% 运算符 Y
\ 除法运算符 带小数的运算会被取整
>>> 无符号右移位操作符 N
其他位移操作符 Y
与 Number 混合运算 N（必须转换为同类型）
Math 对象方法 N
Number 与 BigInt 比较（排序） Y（宽松相等 ==）
Boolean 表现 类型 Number 对象
JSON 中使用 N
